# 01 矩阵总览页 — AI 行业温度计 WebApp（内网版）

> 前置阅读：`00_overview.md`（项目背景、术语、全局规则）  
> 版本：v0（MVP）

---

## 1.1 页面定位

矩阵总览页是本应用的**核心页面**，承载所有数据的可视化展示与快速浏览入口。用户打开应用后默认进入此页面。

---

## 1.2 页面结构

```
┌─────────────────────────────────────────────────────────────┐
│  [固定左上角空白]  │ [横轴] 游戏类型卡片 1 │ 卡片 2 │ 卡片 3 │ → 可水平滚动
├────────────────────┼──────────────────────┼────────┼────────┤
│ [工种大类 A]       │                      │        │        │
│  ├ 工种子类 A1     │  单元格 (A1, GT1)    │  ...   │  ...   │
│  └ 工种子类 A2     │  单元格 (A2, GT1)    │  ...   │  ...   │
├────────────────────┼──────────────────────┼────────┼────────┤
│ [工种大类 B]       │                      │        │        │
│  └ 工种子类 B1     │  单元格 (B1, GT1)    │  ...   │  ...   │
└─────────────────────────────────────────────────────────────┘
```

### 横轴（顶部）

- 展示所有游戏类型卡片，按 `sort_order` 排序
- 支持**水平滚动**（游戏类型数量较多时）
- 每张游戏类型卡片展示：
  - 游戏类型名称
  - **示例海报**（多张并排展示，数量过多时允许图片重叠）
  - 拖拽排序手柄（支持在横轴上拖拽调整顺序，拖拽结束自动保存）
- 点击游戏类型卡片：进入该游戏类型的编辑页（或打开编辑弹窗）

### 纵轴（左侧）

- 展示工种两级列表，按 `sort_order` 排序
- **一级（工种大类）**：作为分组标题行显示，支持**折叠 / 展开**（建议默认展开）
- **二级（工种子类）**：每条子类作为一行，参与与横轴的交叉

### 主体区域（矩阵网格）

- 由横轴 × 纵轴的交叉形成若干**单元格**
- 每个单元格的渲染规则见 1.3 节

---

## 1.3 单元格展示规则

### 有工具卡片时

- 显示**成熟度颜色块**（颜色编码见 `00_overview.md` 0.8 节）
- 显示**工具名称**文字
- 悬停（Hover）时显示编辑 / 删除操作入口（或使用右键菜单，技术选一种保持统一）
- 点击卡片：打开**工具详情弹窗（Modal）**（见 `04_tool_cell_management.md`）

### 无工具卡片时（灰色缺失）

- 显示**灰色占位块**
- v0 统一语义：缺失/暂时不存在，不区分具体原因
- 显示**"添加工具"入口**（v0 全员可见，无鉴权区分）
- 点击"添加工具"入口：打开工具卡片新增表单

---

## 1.4 交互入口汇总

| 操作 | 触发方式 | 跳转/行为 |
|------|----------|-----------|
| 查看工具详情 | 点击有卡片的单元格 | 打开工具详情 Modal |
| 添加工具卡片 | 点击灰色单元格的"添加工具" | 打开工具卡片新增表单（含游戏类型和工种预填） |
| 编辑工具卡片 | Hover 单元格 → 点击编辑 | 打开工具卡片编辑表单 |
| 删除工具卡片 | Hover 单元格 → 点击删除 | 确认后软删除，单元格变为灰色 |
| 拖拽游戏类型排序 | 拖拽横轴游戏类型卡片 | 拖拽结束后调用排序接口持久化 |
| 编辑游戏类型 | 点击横轴游戏类型卡片 | 进入游戏类型编辑页 |
| 折叠/展开工种大类 | 点击大类标题行 | 切换该大类下子类行的显示/隐藏 |

---

## 1.5 游戏类型卡片上的海报展示

- 每张游戏类型卡片上**直接展示示例海报**（多张并排）
- 展示规格：统一裁剪为固定比例缩略图（技术可选 CSS object-fit 或后端生成缩略图）
- 当海报数量过多（超出卡片宽度）时，**允许图片重叠**展示（如扇形叠放或 z-index 堆叠）
- 海报仅作展示用途；上传/删除/排序管理操作入口在游戏类型编辑页内（见 `02_game_type_management.md`）

---

## 1.6 页面级操作入口（建议）

矩阵总览页顶部/导航区建议提供以下快捷入口：

- 前往"游戏类型管理"页
- 前往"工种管理"页
- 前往"维度体系管理"页

---

## 1.7 数据加载说明

- 页面初始化时，一次性加载所有数据：
  - 所有游戏类型（含海报路径）
  - 所有工种大类与子类
  - 所有工具卡片（`GET /tool-cells`）
- 数据量 v0 预期较小（内网维护场景），无需分页或懒加载
- 前端根据数据自行渲染矩阵网格

---

## 1.8 边界与异常

| 情况 | 处理方式 |
|------|----------|
| 无任何游戏类型 | 横轴区域显示空白，提示"尚未添加游戏类型" |
| 无任何工种 | 纵轴区域显示空白，提示"尚未添加工种" |
| 游戏类型或工种均为空 | 主体矩阵区域显示引导文案 |
| 工具卡片删除后 | 单元格立即恢复为灰色占位（前端状态更新） |
