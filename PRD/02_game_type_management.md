# 02 游戏类型管理 — AI 行业温度计 WebApp（内网版）

> 前置阅读：`00_overview.md`（项目背景、术语、全局规则）  
> 版本：v0（MVP）

---

## 2.1 功能概述

游戏类型（Game Type）是矩阵**横轴**的基本单位。本模块负责游戏类型的全生命周期管理：

- 基础 CRUD（增删改查）
- 横轴拖拽排序
- 多维度组合结构化存储（维度体系可配置）
- 示例海报管理（上传/删除/拖拽排序）

---

## 2.2 游戏类型数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | PK | 是 | 主键 |
| `name` | TEXT | **是** | 游戏类型最终展示名称（例如"2.5D 俯视角 RPG"） |
| `sort_order` | INT | 是 | 拖拽排序序号，决定横轴展示顺序 |
| 维度取值关联 | 关联表 | 否 | 通过 `game_type_dimension_value` 关联多个维度取值（可为空） |
| `created_at` / `updated_at` | DATETIME | 是 | 审计时间字段 |
| `created_by` / `updated_by` | TEXT | 是 | 审计用户字段（v0 填 `"system"` 或置空） |
| `is_deleted` | BOOL | 是 | 软删除标记，默认 false |
| `deleted_at` | DATETIME | 否 | 软删除时间 |

---

## 2.3 CRUD 功能

### 2.3.1 新增游戏类型

**入口**：游戏类型管理页的"新增"按钮。

**表单字段**：
- 最终名称（必填）
- 维度取值选择（可选，多选；从"维度体系"中选择，见 `05_dimension_management.md`）
  - 若维度体系尚未配置，此处留空，后续可补填
  - 展示形式：按维度分组，每个维度下可选取一个或多个取值

**保存逻辑**：
1. 校验名称不为空
2. 写入 `game_type` 表，`sort_order` 默认追加到末尾（当前最大值 + 1）
3. 若有维度取值选择，写入 `game_type_dimension_value` 关联记录

### 2.3.2 编辑游戏类型

**入口**：游戏类型管理页的"编辑"操作，或点击矩阵总览页横轴卡片。

**可编辑字段**：
- 最终名称
- 维度取值（覆盖式更新：先删除旧关联记录，再写入新关联记录）

### 2.3.3 删除游戏类型

**删除规则（阻止级联删除）**：

1. 检查该游戏类型是否存在关联数据：
   - 是否有未软删除的 `tool_cell`（关联此 `game_type_id`）
   - 是否有未软删除的 `game_type_poster`（关联此 `game_type_id`）
2. **若存在任一关联数据**：
   - **阻止删除**
   - 向用户展示提示信息，例如：
     - "该游戏类型下存在 N 个工具卡片，请先删除关联工具卡片后再删除此游戏类型。"
     - "该游戏类型下存在 N 张海报，请先删除关联海报后再删除此游戏类型。"
3. **若无关联数据**：
   - 执行软删除：`is_deleted = true`，写入 `deleted_at`
   - 同时软删除关联的 `game_type_dimension_value` 记录（即使为空也统一处理）

---

## 2.4 拖拽排序

- **支持在游戏类型管理页和矩阵总览页横轴上**进行拖拽排序
- 拖拽结束后，调用批量排序接口 `PUT /game-types/sort`，持久化新的 `sort_order` 序列
- 排序接口接收完整的 id 顺序数组，服务端批量更新所有 `sort_order` 值

---

## 2.5 多维度组合结构（维度体系集成）

### 背景

游戏类型名称（如"2.5D 俯视角 RPG"）本质上是多个维度组合的结果。v0 起对维度信息进行**结构化存储**，便于未来进行筛选/聚合。

### 规则

- 每个游戏类型的最终名称（`name`）**始终独立存储**，不由维度自动生成
- 维度取值是附加的结构化信息，可为空（允许只填名称，不挂维度）
- 维度体系本身独立管理（见 `05_dimension_management.md`），可随时新增维度和取值
- 唯一性约束：同一游戏类型不能关联同一维度取值两次（`UNIQUE(game_type_id, dimension_value_id)`）

### 编辑时交互

- 维度取值区域按维度分组展示（例如"玩法：[动作] [RPG]"、"画风：[像素]"）
- 用户可自由勾选/取消，保存时覆盖更新
- 若某维度下无取值可选（维度体系为空），该维度区域可折叠或提示"暂无可选取值"

## 2.6 示例海报管理

### 2.6.1 概述

每个游戏类型可关联**多张**示例海报，用于在矩阵总览页横轴卡片上展示参考图片。

### 2.6.2 海报数据字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | PK | 是 | 主键 |
| `game_type_id` | FK | 是 | 所属游戏类型 |
| `file_path` | TEXT | 是 | 文件在服务器上的存储路径或可访问 URL |
| `sort_order` | INT | 是 | 海报展示顺序，支持拖拽排序 |
| `created_at` / `updated_at` | DATETIME | 是 | 审计时间 |
| `created_by` / `updated_by` | TEXT | 是 | 审计用户（v0 填 `"system"` 或置空） |
| `is_deleted` | BOOL | 是 | 软删除标记，默认 false |
| `deleted_at` | DATETIME | 否 | 软删除时间 |

### 2.6.3 上传海报

**入口**：游戏类型编辑页内的"上传海报"区域。

**规则**：
- 支持单次或多次上传（可多选文件）
- v0 **不限制**文件格式、尺寸、大小，但需在上传区域提供前端提示文案（例如"建议上传 16:9 比例图片，文件大小不超过 5MB"）
- 上传成功后，文件保存至服务器本地目录（例如 `/data/uploads/posters/`），`file_path` 写入数据库
- 新上传的海报默认追加到现有海报列表末尾（`sort_order` 最大值 + 1）

### 2.6.4 删除海报

**规则**：
- 海报是独立实体，删除时**无级联依赖**，直接执行软删除
- 同时需要从服务器本地目录删除（或标记为待清理），避免孤立文件积累
- 软删除：`is_deleted = true`，写入 `deleted_at`

### 2.6.5 海报拖拽排序

- 在游戏类型编辑页内，支持对海报列表进行**拖拽排序**
- 拖拽结束后，调用排序接口 `PUT /posters/sort`（或 `PUT /game-types/{id}/posters/sort`），持久化新的 `sort_order`

### 2.6.6 海报展示（前端）

- 在游戏类型编辑页：展示海报列表，每张海报显示缩略图，提供删除和拖拽手柄
- 在矩阵总览页横轴卡片上：
  - 多张海报并排展示（固定比例缩略图，统一裁剪）
  - 当海报数量过多超出卡片宽度时，**允许图片重叠**（如 z-index 堆叠）
  - 海报的缩略图裁剪方式：前端 CSS（`object-fit: cover`）或后端生成，技术自选

## 2.7 游戏类型管理页面结构

```
游戏类型管理页
├── 顶部操作栏
│   └── [新增游戏类型] 按钮
├── 游戏类型列表（支持拖拽排序）
│   ├── 游戏类型卡片 1
│   │   ├── 名称
│   │   ├── 维度取值标签（若有）
│   │   ├── 海报缩略图预览（若有）
│   │   └── [编辑] [删除] 操作
│   ├── 游戏类型卡片 2
│   └── ...
└── 新增/编辑 表单（弹窗或嵌入式）
    ├── 名称（必填）
    ├── 维度取值选择（可选，分组多选）
    └── 海报管理区域
        ├── 已上传海报列表（可拖拽排序，可删除）
        └── 上传按钮（附提示文案）
```

## 2.8 接口依赖

详细接口定义见 `07_api_design.md`，本模块相关接口：

- `GET /game-types` — 获取游戏类型列表
- `POST /game-types` — 新增游戏类型
- `PUT /game-types/{id}` — 编辑游戏类型
- `DELETE /game-types/{id}` — 软删除游戏类型（含级联检查）
- `PUT /game-types/sort` — 批量更新排序
- `POST /game-types/{id}/posters` — 上传海报
- `DELETE /posters/{poster_id}` — 删除海报（软删除）
- `PUT /posters/sort`（或 `PUT /game-types/{id}/posters/sort`）— 海报排序
- `PUT /game-types/{id}/dimension-values` — 设置游戏类型维度取值（覆盖式）
