# 04 单元格工具卡片管理 — AI 行业温度计 WebApp（内网版）

> 前置阅读：`00_overview.md`（项目背景、术语、全局规则）  
> 版本：v0（MVP）

---

## 4.1 功能概述

工具卡片（Tool Card）存储在**单元格（Cell）**中，单元格是"游戏类型 × 工种子类"的交叉点。

核心规则：
- 每个单元格最多存在 **0 或 1** 个工具卡片（v0 固定，不支持多个）
- 0 = 灰色缺失状态；1 = 有工具卡片，显示成熟度颜色 + 工具名称
- 工具卡片点击后以**弹窗（Modal）**形式展示详情

---

## 4.2 数据字段（`tool_cell` 表）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | PK | 是 | 主键 |
| `game_type_id` | FK | **是** | 关联的游戏类型 |
| `role_id` | FK | **是** | 关联的工种子类（二级） |
| `tool_name` | TEXT | **是** | 工具名称 |
| `maturity_level` | INT | **是** | 成熟度等级，取值 1–5（见 4.3 节） |
| `official_url` | TEXT | **是** | 工具官网链接 |
| `short_desc` | TEXT | **是** | 工具简短描述 |
| `report_url` | TEXT | 否 | 工具报告链接（可选） |
| `created_at` / `updated_at` | DATETIME | 是 | 审计时间 |
| `created_by` / `updated_by` | TEXT | 是 | 审计用户（v0 填 `"system"` 或置空） |
| `is_deleted` | BOOL | 是 | 软删除标记，默认 false |
| `deleted_at` | DATETIME | 否 | 软删除时间 |

**唯一性约束（关键）**：

```sql
UNIQUE(game_type_id, role_id)
```

确保同一单元格最多 1 条有效记录（软删除的记录不计入唯一性约束，需用 partial index 或业务层逻辑处理）。

## 4.3 成熟度等级与颜色编码

### 档位定义

| `maturity_level` | 语义 | 前端颜色 |
|------|------|------|
| 1 | AI 结合程度低 | 红（Red） |
| 2 | 较低 | 橙（Orange） |
| 3 | 中 | 黄（Yellow） |
| 4 | 较高 | 黄绿（Yellow-Green） |
| 5 | AI 结合程度高 | 绿（Green） |
| — | 无工具卡片 | 灰（Gray，非 maturity_level 值，由前端判断单元格为空时使用） |

### 实现约束

- 数据库存储 `maturity_level` 整数（1–5）
- **颜色映射由前端实现**，不存入数据库，避免未来改色需要数据迁移
- v0 颜色直接在前端代码中写死映射即可（如需可维护，可使用 `maturity_config` 表，见 `06_data_design.md`）

## 4.4 CRUD 操作

### 4.4.1 添加工具卡片

**触发条件**：单元格当前无工具卡片（灰色状态）。

**入口**：矩阵总览页灰色单元格上的"添加工具"按钮。

**表单字段**：
- 工具名称（必填）
- 成熟度等级（必填，单选，展示为颜色选择器或下拉列表）
- 工具官网链接（必填）
- 工具简短描述（必填）
- 工具报告链接（可选）

**预填逻辑**：
- 游戏类型和工种子类由点击的单元格位置自动确定，表单中只读展示，不可修改

**保存逻辑**：
1. 校验必填字段不为空
2. 校验 `game_type_id + role_id` 组合不存在未软删除的已有记录（防止唯一约束冲突）
3. 写入 `tool_cell` 表

### 4.4.2 编辑工具卡片

**触发条件**：单元格已有工具卡片。

**入口**：矩阵总览页有卡片单元格 Hover 时出现的"编辑"操作。

**可编辑字段**：
- 工具名称
- 成熟度等级
- 工具官网链接
- 工具简短描述
- 工具报告链接

**不可修改字段**：
- 游戏类型（`game_type_id`）
- 工种子类（`role_id`）

**保存逻辑**：
- 更新 `tool_cell` 记录对应字段，更新 `updated_at` / `updated_by`

### 4.4.3 删除工具卡片

**触发条件**：单元格已有工具卡片。

**入口**：矩阵总览页有卡片单元格 Hover 时出现的"删除"操作。

**删除规则**：
- 工具卡片无下游依赖，**直接执行软删除**，无需级联检查
- 软删除：`is_deleted = true`，写入 `deleted_at`
- 删除后该单元格恢复为灰色缺失状态

## 4.5 工具详情弹窗（Modal）

### 4.5.1 触发方式

点击矩阵总览页中**有工具卡片的单元格**（颜色块区域）。

### 4.5.2 弹窗展示内容

| 内容项 | 展示形式 | 来源字段 |
|--------|----------|----------|
| 工具名称 | 标题文字 | `tool_name` |
| 所在游戏类型 | 只读文字 | 关联 `game_type.name` |
| 所在工种子类 | 只读文字 | 关联 `role.name` |
| 成熟度等级 | 颜色色块 + 文字标签（例如"较高"） | `maturity_level` → 前端映射 |
| 工具官网链接 | 可点击超链接，新标签页打开 | `official_url` |
| 工具简短描述 | 文字段落 | `short_desc` |
| 工具报告链接 | 可点击超链接（若有） | `report_url`（可选，无则不展示） |

### 4.5.3 弹窗操作

- 弹窗内提供**编辑**快捷入口，点击后切换到编辑表单（可为弹窗内表单切换，或关闭后打开编辑弹窗）
- 提供**关闭**按钮

## 4.6 单元格状态机

```
[无卡片 - 灰色]
    │
    ├─ 点击"添加工具" → 填写表单 → 保存
    │                                   │
    ▼                                   ▼
[有卡片 - 颜色块]  ◄──────────────────────
    │
    ├─ 点击卡片 → 打开详情 Modal
    ├─ Hover → 编辑 → 修改并保存
    └─ Hover → 删除 → 确认 → 软删除
                                │
                                ▼
                          [无卡片 - 灰色]
```

## 4.7 接口依赖

详细接口定义见 `07_api_design.md`，本模块相关接口：

- `GET /tool-cells` — 获取全部工具卡片（矩阵渲染用）
- `POST /tool-cells` — 新增工具卡片
- `PUT /tool-cells/{id}` — 编辑工具卡片
- `DELETE /tool-cells/{id}` — 软删除工具卡片
