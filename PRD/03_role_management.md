# 03 工种管理 — AI 行业温度计 WebApp（内网版）

> 前置阅读：`00_overview.md`（项目背景、术语、全局规则）  
> 版本：v0（MVP）

---

## 3.1 功能概述

工种（Role）是矩阵**纵轴**的基本单位，具有**两级层级**：

- **Level 1：工种大类（RoleGroup）** — 纵轴分组标题，例如"2D 美术"、"3D 美术"、"动画与动捕"
- **Level 2：工种子类（Role）** — 实际参与矩阵交叉的行，例如"2D 原画"、"3D 建模"、"动作捕捉"

本模块负责两级工种的全生命周期管理：CRUD、拖拽排序（限组内）。

## 3.2 数据字段

### 3.2.1 工种大类（RoleGroup）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | PK | 是 | 主键 |
| `name` | TEXT | **是** | 大类名称，例如"2D 美术" |
| `sort_order` | INT | 是 | 大类在纵轴上的排列顺序 |
| `created_at` / `updated_at` | DATETIME | 是 | 审计时间 |
| `created_by` / `updated_by` | TEXT | 是 | 审计用户（v0 填 `"system"` 或置空） |
| `is_deleted` | BOOL | 是 | 软删除标记，默认 false |
| `deleted_at` | DATETIME | 否 | 软删除时间 |

### 3.2.2 工种子类（Role）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | PK | 是 | 主键 |
| `role_group_id` | FK | **是** | 所属工种大类 |
| `name` | TEXT | **是** | 子类名称，例如"2D 原画" |
| `sort_order` | INT | 是 | 在所属大类内的排列顺序 |
| `created_at` / `updated_at` | DATETIME | 是 | 审计时间 |
| `created_by` / `updated_by` | TEXT | 是 | 审计用户（v0 填 `"system"` 或置空） |
| `is_deleted` | BOOL | 是 | 软删除标记，默认 false |
| `deleted_at` | DATETIME | 否 | 软删除时间 |

## 3.3 工种大类（RoleGroup）CRUD

### 3.3.1 新增工种大类

**入口**：工种管理页顶部"新增大类"按钮。

**表单字段**：
- 大类名称（必填）

**保存逻辑**：
- 写入 `role_group` 表，`sort_order` 默认追加到末尾（当前最大值 + 1）

### 3.3.2 编辑工种大类

**入口**：工种管理页大类旁的"编辑"操作。

**可编辑字段**：
- 大类名称

### 3.3.3 删除工种大类

**删除规则（阻止级联删除）**：

1. 检查该大类是否存在**未软删除的工种子类**（`role.role_group_id = 此大类 id`）
2. **若存在子工种**：
   - **阻止删除**
   - 提示信息例如："该大类下存在 N 个工种子类，请先删除所有子工种后再删除此大类。"
3. **若无子工种**：
   - 执行软删除：`is_deleted = true`，写入 `deleted_at`

## 3.4 工种子类（Role）CRUD

### 3.4.1 新增工种子类

**入口**：工种管理页某大类旁的"添加子工种"按钮（在大类行内）。

**表单字段**：
- 子类名称（必填）
- 所属大类（自动关联，不需用户选择，取决于点击的是哪个大类的添加入口）

**保存逻辑**：
- 写入 `role` 表，`role_group_id` 为当前大类 id
- `sort_order` 默认追加到该大类内末尾（当前大类内最大值 + 1）

### 3.4.2 编辑工种子类

**入口**：工种管理页子类旁的"编辑"操作。

**可编辑字段**：
- 子类名称
- **不支持修改所属大类**（v0 子类归属不可跨组变更，只能在组内管理）

### 3.4.3 删除工种子类

**删除规则（阻止级联删除）**：

1. 检查该子类是否存在**未软删除的工具卡片**（`tool_cell.role_id = 此子类 id`）
2. **若存在关联工具卡片**：
   - **阻止删除**
   - 提示信息例如："该工种子类下存在 N 个工具卡片，请先删除关联工具卡片后再删除此工种。"
3. **若无关联工具卡片**：
   - 执行软删除：`is_deleted = true`，写入 `deleted_at`

## 3.5 拖拽排序

### 3.5.1 大类排序

- 支持在工种管理页对**工种大类**进行拖拽排序
- 拖拽结束后，调用 `PUT /role-groups/sort` 批量更新 `sort_order`
- 矩阵总览页纵轴顺序随之更新

### 3.5.2 子类排序（组内限制）

- 支持在工种管理页对某大类内的**工种子类**进行拖拽排序
- **限制：子类只能在所属大类内拖拽排序**，不允许跨大类移动（即子类的 `role_group_id` 不可通过拖拽改变）
- 拖拽结束后，调用 `PUT /roles/sort` 批量更新该大类内子类的 `sort_order`
- 排序接口需要验证请求中所有 role id 均属于同一 role_group，防止误操作

## 3.6 工种管理页面结构

```
工种管理页
├── 顶部操作栏
│   └── [新增工种大类] 按钮
├── 工种大类列表（支持拖拽排序）
│   ├── 大类 A（可拖拽）
│   │   ├── 大类名称 + [编辑] [删除] + [添加子工种] 操作
│   │   └── 子类列表（支持组内拖拽排序）
│   │       ├── 子类 A1 + [编辑] [删除]
│   │       └── 子类 A2 + [编辑] [删除]
│   ├── 大类 B
│   │   └── ...
│   └── ...
└── 新增/编辑表单（弹窗或嵌入式）
    └── 名称（必填）
```

## 3.7 接口依赖

详细接口定义见 `07_api_design.md`，本模块相关接口：

- `GET /role-groups` — 获取大类列表（含子类，按 `sort_order`）
- `POST /role-groups` — 新增大类
- `PUT /role-groups/{id}` — 编辑大类
- `DELETE /role-groups/{id}` — 软删除大类（含级联检查）
- `PUT /role-groups/sort` — 大类批量排序
- `POST /roles` — 新增子类（需 `role_group_id`）
- `PUT /roles/{id}` — 编辑子类
- `DELETE /roles/{id}` — 软删除子类（含级联检查）
- `PUT /roles/sort` — 子类批量排序（仅组内）
