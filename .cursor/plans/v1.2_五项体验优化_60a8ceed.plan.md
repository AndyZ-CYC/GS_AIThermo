# v1.2 五项体验优化

## 问题 1：ToolCellModal 定位偏移修复

**根因分析**：[App.tsx](frontend/src/App.tsx) 第 41 行的 `.animate-page-in` 使用了带 `transform` 的 CSS 动画（`animation: page-in 250ms ease-out both`），`fill-mode: both` 使最终状态保留 `transform: translateY(0)`。**CSS 规范中，任何设置了 `transform` 的元素会成为 `fixed` 定位子元素的新包含块**，导致 `ToolCellModal` 的 `fixed inset-0` 不是相对于 viewport，而是相对于这个动画容器。

**修复方案**：使用 React `createPortal` 将 `ToolCellModal` 渲染到 `document.body`，绕过 transform 包含块限制。

- 修改 [ToolCellModal.tsx](frontend/src/components/ToolCellModal.tsx)，外层包裹 `createPortal(..., document.body)`

## 问题 2：GameType 新增"类别描述"和"类别示例"字段

### 后端

- **数据库迁移** ([database.py](backend/database.py))：`game_type` 表新增 `description TEXT DEFAULT ''` 和 `examples TEXT DEFAULT '[]'`（JSON 字符串数组），使用 safe migration 模式（检查列是否存在再 ALTER）
- **Pydantic 模型** ([models.py](backend/models.py))：
  - `GameTypeCreate` 增加 `description: str = ""` 和 `examples: list[str] = []`
  - `GameTypeUpdate` 增加相同字段
  - `GameTypeOut` 增加 `description: str` 和 `examples: list[str]`
- **API 路由** ([routers/game_types.py](backend/routers/game_types.py))：
  - `create_game_type`：写入 description + json.dumps(examples)
  - `update_game_type`：更新 description + examples
  - `_build_game_type_out`：读出 examples 时 json.loads 为 list
  - `list_game_types`：同步修改

### 前端

- **类型** ([types/index.ts](frontend/src/types/index.ts))：`GameType` 增加 `description: string` 和 `examples: string[]`
- **API 层** ([api/gameTypes.ts](frontend/src/api/gameTypes.ts))：创建/更新请求体增加新字段
- **Hooks** ([hooks/useGameTypes.ts](frontend/src/hooks/useGameTypes.ts))：同步更新 mutation 参数

## 问题 3：矩阵表头 GameType 卡片 Hover 信息窗口

## 问题 4：数据管理 - GameType 管理卡片改为可展开形式

重构 [GameTypeManagement.tsx](frontend/src/pages/GameTypeManagement.tsx) 中的 `SortableGameTypeCard`：

- **折叠态**（默认）：显示拖拽手柄 + 名称 + 操作按钮（与现有一致）
- **展开态**（点击展开箭头或"编辑"按钮触发）：卡片纵向展开，inline 展示：
  - 名称编辑（input）
  - 类别描述编辑（textarea）
  - 类别示例编辑（tag 输入组件：输入框 + 回车/逗号添加 tag，tag 可删除）
  - 海报管理区域（上传 + 缩略图列表 + 删除）
  - 保存/取消按钮
- 展开/折叠使用 CSS `max-height` + `overflow-hidden` 过渡动画

**新建 Tag 输入子组件**：可在 `SortableGameTypeCard` 内部实现，输入框 + tag 列表，支持回车添加、点击 x 删除。

## 问题 5："不适用"占位灰色卡片

### 后端

- **数据库** ([database.py](backend/database.py))：`tool_cell` 表新增 `is_na INTEGER NOT NULL DEFAULT 0`（safe migration）
- **Pydantic 模型** ([models.py](backend/models.py))：
  - `ToolCellCreate` 增加 `is_na: bool = False`；当 `is_na=True` 时放宽必填校验
  - `ToolCellOut` 增加 `is_na: bool`
- **API 路由** ([routers/tool_cells.py](backend/routers/tool_cells.py))：
  - `create_tool_cell`：当 `is_na=True` 时，`tool_name` 设为 "N/A"，`maturity_score` 设为 0，`official_url` 和 `short_desc` 设为占位值
  - `update_tool_cell`：支持从 N/A 转换为正常卡片（清除 `is_na` 标记 + 填入真实数据）

### 前端

- **类型** ([types/index.ts](frontend/src/types/index.ts))：`ToolCell` 增加 `is_na: boolean`
- **矩阵 CellCard** ([MatrixOverview.tsx](frontend/src/pages/MatrixOverview.tsx))：
  - 当 `cell.is_na` 时渲染灰色卡片，显示"不适用"文字 + 斜杠图标
  - 点击后弹出 Modal，提供"移除标记"和"转换为工具卡片"两个选项
- **ToolCellModal** ([ToolCellModal.tsx](frontend/src/components/ToolCellModal.tsx))：
  - `create` 模式新增一个开关/选项："标记为不适用"
  - 勾选后隐藏其余表单字段，直接提交 `is_na=true` 的记录
- **空格子点击**：弹窗提供两个入口 -- "添加工具" 和 "标记为不适用"
