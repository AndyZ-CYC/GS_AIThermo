# v1.1 前端体验修正计划

## 修正 1：合并「游戏类型管理」与「工种管理」为双栏单页

**问题**：两个管理页面各自 `max-w-3xl` 偏左，右侧大量留白。

**方案**：

- 将路由从三页改为两页：矩阵总览 `/` + 数据管理 `/manage`
- 修改 [frontend/src/App.tsx](frontend/src/App.tsx)：导航项从 3 个改为 2 个（"矩阵总览"、"数据管理"），新增 `DataManagement` 路由
- 新建 [frontend/src/pages/DataManagement.tsx](frontend/src/pages/DataManagement.tsx)：左右双栏布局（`grid grid-cols-2 gap-6`），左栏为游戏类型管理，右栏为工种管理
- 原有 [frontend/src/pages/GameTypeManagement.tsx](frontend/src/pages/GameTypeManagement.tsx) 和 [frontend/src/pages/RoleManagement.tsx](frontend/src/pages/RoleManagement.tsx) 的内容提取为组件嵌入，去掉 `max-w-3xl` 限制，改为 `w-full`
- 也可选择直接把两个文件的导出组件合并到 DataManagement 中（减少文件数），保留原文件作为子组件导出

## 修正 2：全局字号上调

**问题**：当前大部分文字使用 `text-xs`（12px）和 `text-sm`（14px），可读性不足。

**方案**（涉及多个文件的 className 修改）：

- 矩阵卡片中工具名称：`text-xs` -> `text-sm`（[MatrixOverview.tsx](frontend/src/pages/MatrixOverview.tsx) 第 333 行）
- 矩阵页统计摘要：`text-sm` -> `text-base`
- 矩阵页图例标签：`text-xs` -> `text-sm`
- 导航栏链接：`text-sm` -> `text-base`（[App.tsx](frontend/src/App.tsx) 第 27 行）
- 工种大类标签和子类名称：`text-sm` -> `text-base`
- 管理页标题：`text-xl` -> `text-2xl`
- 管理页列表项文字：`text-sm` -> `text-base`
- 管理页按钮文字：`text-xs` -> `text-sm`

## 修正 3：矩阵子工种名称颜色提亮

**问题**：当前子工种名称使用 `text-text-secondary`（`#8a8a95`），在暗色背景上可读性差。

**方案**：

- [MatrixOverview.tsx](frontend/src/pages/MatrixOverview.tsx) 第 237 行的 `RoleRow` 组件中：
  - `text-text-secondary` -> `text-text-primary` 或使用一个中间值如 `text-text-primary/80`（约 `rgba(240,240,242,0.8)`）

## 修正 4：增强背景光源/渐变效果

**问题**：当前 body 渐变的透明度太低（`0.035` / `0.015`），视觉上几乎不可见。

**方案**：
修改 [frontend/src/index.css](frontend/src/index.css) 中 body 的 `background-image`：

- 左上角暖白光源：`rgba(255, 250, 240, 0.035)` -> `rgba(255, 250, 240, 0.07)` 并扩大范围到 `70%`
- 右下角橙金光源：`rgba(232, 145, 58, 0.015)` -> `rgba(232, 145, 58, 0.04)` 并扩大范围到 `60%`
- 可额外增加一层极淡的中心光晕：`radial-gradient(ellipse at 50% 30%, rgba(255,255,255,0.02), transparent 50%)`

## 修正 5：增大游戏海报尺寸 + 无海报时文字居中

**问题**：海报 `w-7 h-7`（28px）太小；无海报时 `justify-end` 使文字偏下。

**方案**：
修改 [MatrixOverview.tsx](frontend/src/pages/MatrixOverview.tsx) 中 `GameTypeHeader` 组件：

- 游戏类型表头整体高度增加：`py-3` -> `py-4`
- 海报图片尺寸：`w-7 h-7` -> `w-12 h-12`（48px），更清晰可见
- 溢出指示器同步：`w-7 h-7` -> `w-12 h-12`
- 无海报时布局对齐：`justify-end` -> `justify-center`（改为垂直居中）
- 有海报时保留 `justify-end`（文字在上，海报在下），所以条件判断：
  - 有海报：`justify-between`（名称顶部，海报底部）
  - 无海报：`justify-center`（名称居中）

## 修正 6：一级工种类别行横向滚动时固定

**问题**：当矩阵横向滚动时，工种一级大类（`RoleGroupRows` 的 group header）随内容一起滚走，不像二级子工种名称那样 `sticky left-0`。

**方案**：
修改 [MatrixOverview.tsx](frontend/src/pages/MatrixOverview.tsx) 中 `RoleGroupRows` 的 group header div：

当前实现使用 `gridColumn: 1 / colCount+1` 让它跨满，但这导致它整体随滚动移动。解决方案是让 group header 的**内容**具有 sticky 定位：

```
<div style={{ gridColumn: `1 / ${colCount + 1}` }} className="relative">
  <div className="sticky left-0 w-fit ...">
    ▼ 工种大类名称 (N 个子工种)
  </div>
</div>
```

外层 div 仍然跨满提供背景色，内部 div `sticky left-0 w-fit` 使文字始终贴左。

## 修正 7：页面切换动效

**问题**：当前路由切换无任何过渡效果，显得生硬。

**方案**：
不引入新依赖，使用 CSS animation + `useLocation().pathname` 作为 key 触发重新挂载动画：

修改 [frontend/src/App.tsx](frontend/src/App.tsx)：

- 在 `<main>` 内部包裹一个 `<div key={location.pathname} className="animate-page-in">` 
- 在 [frontend/src/index.css](frontend/src/index.css) 中定义 `@keyframes page-in`：从 `opacity: 0; transform: translateY(6px)` 到 `opacity: 1; transform: translateY(0)`，时长 250ms
- 定义对应的 `.animate-page-in` 工具类
